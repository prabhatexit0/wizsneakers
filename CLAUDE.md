# Wizsneakers — Claude Code Project Guide

## What This Is
A 2D top-down pixel art RPG (Pokemon-style) where sneaker culture meets creature collection. Built with **Rust/WASM** (game engine) + **React/TypeScript/Vite** (frontend).

## Tech Stack
- **Engine**: Rust → WebAssembly via `wasm-pack` + `wasm-bindgen` + `serde`
- **Frontend**: React 18+, TypeScript 5+, Vite 8+
- **Rendering**: HTML5 Canvas (overworld), React DOM (UI/menus/battle)
- **Audio**: Web Audio API (SFX) + HTMLAudioElement (BGM)
- **Storage**: localStorage (saves)
- **Package Manager**: pnpm (monorepo)

## Architecture
- **Rust/WASM is the single source of truth** for ALL game state. React never mutates game state.
- React-side state is UI-only (menu open, animation progress, text speed).
- One `GameEngine` struct exposed to JS. JSON for complex data, direct getters for hot-path.

## Build Commands
```bash
# Build WASM engine (outputs to client/src/wasm/)
pnpm run build:engine

# Build everything (engine + client)
pnpm run build

# Dev server (builds engine first, then starts Vite)
pnpm run dev

# Run Rust tests
cd engine && cargo test

# TypeScript type check
cd client && npx tsc --noEmit

# Full verification (use this to check everything works)
./verify.sh
```

## Performance Budgets
- 60fps target, WASM tick <0.5ms, Canvas render <8ms
- WASM module <200KB gzipped, total assets <5MB
- Save file <32KB per slot

## PRD Execution
See `prds/00-master.md` for the autonomous execution plan. Each PRD in `prds/` is a self-contained phase with:
- Clear deliverables and files to create/modify
- Rust test requirements that must pass
- A verification command that gates completion

Run `./verify.sh` after each phase to confirm everything passes.
